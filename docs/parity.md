# Parity

Crate-by-crate **presence + depth** report. Presence is regenerated by
`cargo xtask parity`; depth grades come from the 2026-04-30 audit (see
[`audit-2026-04.md`](audit-2026-04.md) and the roadmap in
[`full-port-plan.md`](full-port-plan.md)).

## Depth grading

- **a** — feature-complete vs upstream, full TCK / multi-node coverage.
- **b** — production-functional; minor operators / knobs missing.
- **c** — happy-path implementation; protocol shortcuts; partial TCK.
- **d** — public types present, active machinery missing or stubbed.
- **f** — placeholder / not yet implemented.

## Roadmap

The 15-phase plan in [`full-port-plan.md`](full-port-plan.md) is the
path from the current depth grades to all-`a`/`b`. `cargo xtask
parity` is being upgraded in Phase 0 to surface depth grades
automatically; until then, this table is hand-maintained against the
audit baseline.
