# Alignment surface

A crate-by-crate view of the atomr surface, graded by how deeply each
crate covers the feature shape mature actor runtimes have converged
on. Presence is regenerated by `cargo xtask parity`; depth grades are
maintained against the empirical baseline in
[`audit-2026-04.md`](audit-2026-04.md).

This is an alignment view, not a completion view. As atomr grows past
prior art (see [`actors-and-agentic-computing.md`](actors-and-agentic-computing.md)
for the unified-compute thesis), the table is the place to record
which subsystems extend further than the inherited shape.

## Depth grades

- **a** — feature-complete with full conformance test coverage.
- **b** — production-functional; minor operators / knobs missing.
- **c** — happy-path implementation; protocol shortcuts; partial
  conformance.
- **d** — public types present, active machinery missing or
  stubbed.
- **f** — placeholder, not yet implemented.
